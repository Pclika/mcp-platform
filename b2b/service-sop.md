# ToB 服务交付 SOP

**Pclika MCP Platform — 商业服务标准操作流程 v0.1**

---

## 总流程图

```
询盘接入
  └─ 24h 发送需求确认问卷
       └─ 1工作日 收集回复
            └─ 3工作日 出方案 + 报价单
                 └─ 客户确认 + NDA 签署（如需）
                      └─ 合同签署 + 首付款
                           └─ 项目执行
                                └─ 交付 + 验收
                                     └─ 尾款 + 归档
                                          └─ 售后支持（按合同）
```

---

## SOP-B01：询盘接收与分类

**触发条件：** 收到客户邮件、网站 B2B 表单、社交媒体私信

**执行步骤：**

1. **登记**（30 分钟内）
   - 在 `b2b/crm-log.md`（或 CRM 系统）新建客户记录
   - 记录：来源渠道、公司名、联系人、询盘日期、需求摘要
   - 分配流水号：`PCK-QT-YYYY-NNN`

2. **分类**（1 小时内）
   - 🔴 **紧急**：明确提到项目时限 ≤ 2 周，或大额项目（≥¥100,000）
   - 🟡 **标准**：一般需求，正常处理
   - 🟢 **咨询**：只询问产品信息，无明确采购意向

3. **响应**（24 小时内）
   - 发送确认邮件 + 需求问卷（见 `b2b/inquiry-questionnaire.md`）
   - 邮件模板：

```
主题：Re: [公司名] Pclika MCP Platform 合作咨询

您好 [联系人姓名]，

感谢您对 Pclika 的关注！我们已收到您的询盘，将在 3 个工作日内提供方案建议。

为了帮您更快匹配合适方案，请花 5 分钟填写以下需求问卷：
[附件：inquiry-questionnaire.md] 或 [内联问卷链接]

如有紧急需求，可直接回复邮件说明。

Pclika Team
```

## SOP-B02：方案制定与报价

**触发条件：** 收到客户填写的需求问卷

**执行步骤：**

1. **需求分析**（1 工作日）
   - 对照 SKU 目录（`product/sku-catalog.md`）匹配标准产品
   - 评估定制开发工作量（firmware / schema / PCBA）
   - 确认关键技术点：传感器类型、通信协议、交期要求

2. **方案文档**（1 工作日）
   - 产出：方案说明 PDF（含架构图 + 硬件选型 + 软件方案）
   - 工具：可用 Claude 辅助生成技术方案初稿

3. **报价单**（1 工作日）
   - 填写 `b2b/quote-template.md`，更新编号为 `PCK-QT-YYYY-NNN`
   - 逐项核算成本：BOM + 服务工时 + 包装运费
   - 按定价模型（`product/pricing-model.md`）确认毛利率 ≥ 40%

4. **发送报价**
   - 邮件主题：`Pclika 方案报价 — [项目名] — PCK-QT-YYYY-NNN`
   - 附件：报价单 PDF + 方案说明（如有）
   - 跟进提醒：3 天后若无回复则跟进

## SOP-B03：合同签署与首付款

**触发条件：** 客户口头或邮件确认报价

**执行步骤：**

1. **NDA 签署**（如客户有保密需求）
   - 发送 `b2b/nda-template.md` 填写版本
   - 双方签署后扫描件归档

2. **服务合同**（≥ ¥20,000 项目必须）
   - 基于报价单生成正式服务合同
   - 关键条款：交付物描述、验收标准、付款节点、违约条款

3. **首付款收取**
   - 发送付款信息（银行卡或企业支付宝）
   - 确认到账后，发送开工确认邮件

4. **项目启动**
   - 在项目看板（GitHub Issues / Notion）创建项目卡
   - 分配负责人，设定里程碑节点

## SOP-B04：项目执行

**原型整合服务（SVC-PROTO）标准流程：**

| 阶段 | 内容 | 时间 |
|------|------|------|
| Day 1 | 技术对齐会议（30 分钟视频）、确认最终需求 | 第 1 天 |
| Day 2–3 | 硬件选型与采购确认、固件方案实现 | 第 2–3 天 |
| Day 4 | 功能联调、MCP 工具测试 | 第 4 天 |
| Day 5 | 交付包准备（固件 + 文档 + 配置文件）| 第 5 天 |

**固件定制服务（SVC-FW）标准流程：**

| 阶段 | 内容 |
|------|------|
| 需求冻结 | 书面确认功能列表（不可在中途随意扩大） |
| 开发 | 基于 Pclika Runtime，每日提交进度到私有 Repo |
| 代码审查 | 客户可参与，或由 Pclika 内部 review |
| 交付 | 固件 Binary + 源码（如约定）+ 文档 + 测试报告 |
| 变更 | 超出需求范围的功能按 ¥3,000/天 额外计费 |

## SOP-B05：交付与验收

**交付包内容（标准）：**

- [ ] 固件 Binary（含版本号和 Seal）
- [ ] 烧录说明（idf.py flash 指令 or .bin 直烧）
- [ ] MCP 配置文件（Claude Code / Cursor / Codex 版本各一份）
- [ ] 硬件连接图（接线说明 PDF）
- [ ] API / 工具 Schema 文档
- [ ] 测试报告（已测功能列表 + 测试结果）

**验收流程：**

1. 客户在收到交付物后 **5 个工作日内** 完成验收测试
2. 发现问题提交 Bug 报告，Pclika **3 工作日内** 响应修复
3. 累计最多 **2 轮** 免费修复；第 3 轮起按 ¥1,500/天 计费
4. 客户书面或邮件确认验收通过 → 触发尾款支付

---

## SOP-B06：售后与归档

1. **尾款收取** — 验收确认后 7 工作日内到账
2. **归档** — 项目文件（合同/报价/交付物）归档至 `b2b/projects/PCK-QT-YYYY-NNN/`
3. **满意度回访** — 交付 30 天后发送 3 问满意度邮件
4. **续约提醒** — 年度支持合同到期前 60 天主动联系续签
5. **案例整理** — 获客户授权后，整理为官网案例（脱敏处理）
