# Project Foundation

## What We Are Building

Pclika is building an open-source embedded platform that combines:

- a standard hardware base
- reusable firmware foundations
- MCP-oriented AI development workflows
- guided website-based product and capability selection

The platform is designed so that a developer can import the repository into Claude, Codex, Cursor, OpenCode, or another MCP-compatible tool and immediately begin useful embedded development from a shared technical baseline.

## First Platform Objective

The first objective is not to build every feature at once.

It is to stabilize the core foundation:

- one standard baseboard
- one stable runtime model
- one clear MCP bridge contract
- one modular extension model
- one website path that makes the platform easy to understand and buy

## Hardware Definition Strategy

The hardware foundation should start with a clear scope:

- ESP32-S3 base platform
- sensing-ready
- display-ready
- servo-ready
- future camera-ready
- future industrial-ready

The baseboard should not try to solve every problem. It should provide a clean platform for repeated expansion.

## Extension Strategy

The first extension roadmap should cover:

- Sensor
- Display
- Servo and motion
- Vision
- Industrial communication

Each extension should plug into the same pattern:

- module hardware
- runtime support
- MCP tools
- example projects
- website product path

## Open-Source Strategy

The open-source repository should be the development center of gravity.

That means:

- all core docs live in the repository
- the runtime model is visible and reusable
- MCP tool contracts are documented
- examples are easy to run
- prompt context is available for AI tools

## Website Strategy

The website should act as a guided front door for both people and products.

It should help users:

- understand the platform
- choose the right kit
- choose the right modules
- jump into docs and examples
- request B2B support if needed

## Developer Experience Strategy

The platform should reduce startup time for both humans and AI tools.

A good experience looks like this:

1. import repository
2. read root context
3. choose capability path
4. connect board and module
5. build and flash example
6. iterate with AI support

## Success Metrics

Platform foundation success should be measured by:

- setup clarity
- example success rate
- extension consistency
- AI import usefulness
- speed to first working demo

## Immediate Next Deliverables

The next concrete deliverables after this foundation should be:

- hardware interface map
- runtime capability map
- first MCP bridge implementation plan
- first website IA and content plan
- first example backlog
