MCP Channel · SSE Transport
Pclika MCP Endpoint
Direct Connection — This channel exposes all Pclika hardware tools via the MCP protocol over SSE. Claude, Cursor, VS Code and any MCP-compatible AI client can connect directly — no local bridge required.
SSE Endpoint
https://hc.pclika.com/sse
HTTP Endpoint (Streamable)
https://hc.pclika.com/mcp
Protocol: MCP v1.0 · JSON-RPC 2.0  ·  Transport: SSE / HTTP Streaming  ·  Auth: API Key (header)
Setup · 3 Steps
Quick Start
1
Add to Claude Code
claude mcp add pclikaPlatform \
  --transport sse https://hc.pclika.com/sse \
  --header "X-API-Key: YOUR_API_KEY"
Replace YOUR_API_KEY with your key from the Auth section below.
2
Verify registration
claude mcp list
# → pclikaPlatform  sse  https://hc.pclika.com/sse  ✓
3
Start using hardware tools
# In Claude Code, just type naturally:
Read the temperature sensor
Turn on GPIO pin 4
Show "Hello" on the display
Scan for Wi-Fi networks

Authentication
API Key
All requests to the MCP channel must include your API key in the X-API-Key header. The key is issued per-device and tied to your hardware serial number. 
# Pass as header in claude mcp add
claude mcp add pclikaPlatform \
  --transport sse https://hc.pclika.com/sse \
  --header "X-API-Key: pck_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"

# Or in config JSON
{
  "url": "https://hc.pclika.com/sse",
  "headers": { "X-API-Key": "pck_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx" }
}
Your API key is printed on the kit packaging and available via support. Keys follow the format pck_ + 32 hex characters.
